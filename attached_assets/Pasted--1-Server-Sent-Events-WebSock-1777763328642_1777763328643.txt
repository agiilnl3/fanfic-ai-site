🚀 Новые функции
1. Стриминг текста в реальном времени
Использовать Server-Sent Events или WebSockets, чтобы текст истории появлялся слово-за-словом во время генерации (как в ChatGPT). Это устранит «слепое ожидание» и сделает создание увлекательным.

2. Многоглавные истории и саги
Возможность продолжать историю — «Глава 2», «Глава 3» с сохранением контекста персонажей и мира. Бэкенд хранит характеры/сюжет как «биографию мира».

3. Соавторство с AI
Режим, где автор пишет одно предложение → AI продолжает следующий абзац → автор вмешивается. Чат-подобный интерфейс совместного письма.

4. Озвучка историй (text-to-speech)
Кнопка «Listen» на странице истории — генерация аудио через OpenAI TTS с разными голосами для повествователя и персонажей. Аудиоплеер с подсветкой текущего абзаца.

5. Социальная составляющая
Лайки/закладки, «Continue This Story» (форк чужой истории и развитие в свою сторону), подписки на авторов, лента ленты подписок.

6. Шаблоны и персонажи
Библиотека готовых архетипов («детектив-нуар», «эльфийский маг») и сохранённых пользователем персонажей, которых можно переиспользовать в разных историях для визуальной согласованности.

7. Экспорт в PDF/EPUB
Красиво свёрстанная книга с обложкой, иллюстрациями и оглавлением — для печати или Kindle.

8. Multi-language
Генерация на русском, испанском, японском и др. + интерфейс на нескольких языках через i18n.

🎨 Улучшения UI/UX
9. Темы оформления
Кроме текущей dark indigo/amber — light mode, sepia (как читалка), high-contrast для accessibility. Переключатель в шапке.

10. Настройки чтения
На странице истории: размер шрифта, ширина колонки, межстрочный интервал, выбор шрифта (serif/sans/dyslexic-friendly). Сохранять в localStorage.

11. Превью обложки в Library
Сейчас карточки одинакового размера — добавить hover-эффект с краткой аннотацией, прогресс-баром «X% прочитано», бэйджами популярности.

12. Onboarding для новых пользователей
Первый визит → 3-шаговый тур по интерфейсу + создание первой короткой истории за 30 секунд (с пресетом).

13. Мобильная адаптация
Сейчас работает, но bottom-sheet навигация, swipe-жесты для перелистывания «глав», крупные тач-таргеты для перегенерации.

14. Inline-комментарии автора
Автор может выделить абзац и оставить «заметку для себя» (не видна публично) — для отслеживания идей при редактировании.

⚡ Оптимизация производительности
15. Object Storage для иллюстраций
Сейчас изображения хранятся как base64 в PostgreSQL — это раздувает БД и тормозит запросы. Перенести на App Storage (S3-совместимое), хранить только URL → быстрая лента, меньше трафика.
